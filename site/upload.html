Update GitHub Pages content

Keep the deck browser and inventory table in sync with the Markdown source. Follow these checklists when adding, amending, or removing content so GitHub Pages always reflects the latest lists.

Markdown-first No build step Dark mode ready
Decks

Add a new deck

  • Create or import a Markdown file in decks/ (e.g., mtg-decks create "New Deck" "Commander" --colors W U).
  • Open decks.html and append a deck entry to the decks array with id, title, commander, and the Markdown filename.
  • Commit the new decks/<deck>.md file and the updated decks.html, then push so GitHub Pages can serve it immediately.
Deck lives in Markdown Listed in decks array

Amend an existing deck

  • Edit the corresponding Markdown in decks/ and save.
  • If you renamed the file, update its file path and title in the decks array inside decks.html.
  • Re-run valuation or spec checks if needed, commit the changes, and push.
Edit markdown directly Update paths if renamed

Remove a deck

  • Delete the Markdown file from decks/.
  • Remove its entry from the decks array in decks.html so the browser list stays clean.
  • Commit both removals and push to update GitHub Pages.
Clean browser list Remove stale files
Inventory

Add or update cards

  • Use mtg-decks spares import --box "Box Name" with a CSV or pasted list to merge counts into inventory.md. Sorting flags rewrite the table for you.
  • Commit the refreshed inventory.md so the inventory table reflects the latest counts.
  • Open inventory.html locally or on Pages to verify the table renders as expected.
CLI rewrites table Box + type filters

Remove cards

  • Run another mtg-decks spares import with updated counts (or a CSV/text file that omits removed cards) to regenerate inventory.md.
  • Alternatively, open inventory.md and delete the rows you no longer need, then save.
  • Commit the updated inventory.md and push to refresh GitHub Pages.
Regenerate table Manual edits allowed

Ship it

After any deck or inventory change, run pytest -q locally if you have the tooling installed, then commit and push. GitHub Pages will serve the updated Markdown files immediately.

Need the CLI? Run pip install -e . from the repo root to install the local tools.
